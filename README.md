# CDLPassPrep

Marketing site + growth funnel for the CDLPassPrep digital bundle (17 CDL study guides +
browser-based practice quiz trainer), built with Next.js 14 (App Router) + Tailwind,
deployed on Vercel, with Stripe Checkout for payment, Supabase for storage/leads/events,
and Resend for transactional + drip email.

## Getting started

```bash
npm install
npm run dev
```

The site runs and looks correct with **no environment variables set** — checkout,
downloads, email capture, and analytics all fail gracefully (friendly errors or silent
no-ops) instead of crashing. Fill in the env vars below to make everything fully live.

## Environment variables

Create a `.env.local` file (never commit this) with:

```
# Stripe — from https://dashboard.stripe.com/apikeys
STRIPE_SECRET_KEY=sk_live_or_test_...

# Optional: create a fixed Price in the Stripe dashboard and set this to use it for
# checkout instead of the inline price_data fallback. Lets you A/B or change price
# without touching code — pair with NEXT_PUBLIC_PRICE_DISPLAY below.
STRIPE_PRICE_ID=price_...

# Display price shown across the site (defaults to "99" if unset). Change this to
# test a different launch price — just make sure it matches your Stripe Price amount.
NEXT_PUBLIC_PRICE_DISPLAY=99

# Supabase — private bucket holding the guide PDFs, plus leads/events tables
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJ...   # service role key — server-side only, never expose to the client

# The deployed site URL (used to build Stripe redirect URLs, sitemap, emails)
NEXT_PUBLIC_SITE_URL=https://cdlpassprep.com

# Resend — from https://resend.com/api-keys
RESEND_API_KEY=re_...
EMAIL_FROM=CDLPassPrep <hello@cdlpassprep.com>

# Shared secret protecting the drip-email cron endpoint. Vercel Cron sends this
# automatically as an Authorization header when CRON_SECRET is set as an env var —
# also used to derive the unsubscribe-link token, so keep it stable once emails go out.
CRON_SECRET=some-long-random-string
```

On Vercel, add these under Project Settings → Environment Variables.

## Database (Supabase)

Three tables, defined in `supabase/migrations/`:

- `0001_leads.sql` — `leads` table: everyone who opts in via the quiz, `/free`, or
  exit-intent. Powers the email drip. Creates it with RLS enabled (no policies — deny by
  default for anon/authenticated; the app only ever accesses it via the service_role key,
  which bypasses RLS).
- `0002_lead_emails_and_events.sql` — `lead_emails` (prevents duplicate drip sends) and
  `events` (lightweight, no-PII funnel analytics), same RLS approach.
- `0003_grants.sql` — explicit `GRANT` statements for `service_role` on all three tables.
  Needed because enabling RLS via the Supabase SQL Editor's "Run and enable RLS" prompt
  can leave the service_role key without read/write privileges even though it's meant to
  bypass RLS — run this if you see `permission denied for table ...` errors.

Run these in order in the Supabase SQL Editor (dashboard → SQL Editor → paste → run)
against your project. The site works without them — inserts fail gracefully and log a
warning — but nothing persists until the tables exist with the right grants.

## Payment flow

1. Visitor clicks any "Get Instant Access" button → `CTAButton` component fires a
   `checkout_click` analytics event, then posts to `/api/checkout`.
2. `/api/checkout` creates a Stripe Checkout Session — using `STRIPE_PRICE_ID` if set,
   otherwise building the line item inline from `PRICE_USD_CENTS` (derived from
   `NEXT_PUBLIC_PRICE_DISPLAY`).
3. On success, Stripe redirects to `/success?session_id={CHECKOUT_SESSION_ID}`.
4. `/success` verifies the session server-side via the Stripe API before showing
   anything — never trusted purely from the URL.
5. Once verified, `lib/supabase.ts` generates a signed, 24-hour-expiring URL for the
   guide bundle ZIP, and the page also links to `/trainer` (see below).
6. Stripe's built-in email receipt goes out automatically; the success page URL is safe
   to reuse as a "re-open my downloads" link.

## Uploading the actual guide files

1. In Supabase, create a **private** Storage bucket named `cdlpassprep-downloads` (or
   update `DOWNLOADS_BUCKET` in `lib/supabase.ts` if you name it differently).
2. Upload the bundle ZIP and update the `path` value in `DOWNLOAD_FILES` in
   `lib/supabase.ts` to match.
3. Never make the bucket public — signed URLs are how buyers get access.

## Practice quiz trainer (paid, `/trainer`)

`public/trainer/` is a static build of a separate Vite + React app (originally an
Electron desktop app; the same source builds as a plain web app with no changes). It's
served at `/trainer` via a rewrite in `next.config.mjs`, and linked from the success
page. It is **not** access-gated — the URL isn't published anywhere except the success
page, but anyone with the link can reach it. That's an intentional simplicity tradeoff;
add real gating (e.g. a purchase token check) if that's not good enough. To ship an
updated question set, rebuild the trainer source and copy its `dist/*` contents into
`public/trainer/` again — no need to redistribute anything to past buyers.

## Free practice quiz (`/quiz`) — the growth funnel

The free, public-facing 20-question quiz — separate from the paid `/trainer` app above.

- `data/quiz-questions.ts` — a 60-question bank, original writing anchored to public
  AAMVA/FMCSA material, tagged by category. `sampleQuizQuestions()` picks a shuffled 20
  per attempt (8 general, 4 air-brakes, 3 combination, 3 pre-trip, 2 road-signs-safety)
  with shuffled answer order too.
- `components/quiz/QuizApp.tsx` — client-side reducer state machine: intro → quiz →
  score (email-gated detailed breakdown, with a skip option) → results (category bars,
  study plan if opted in, per-question review, bundle pitch keyed to weakest category).
- No email required to take the test or see per-question results — only the detailed
  weak-area breakdown + study plan are gated, and even skipping that still shows every
  answer. Never fully hostage the basics behind an email wall.

## Email capture + drip (Resend)

- `POST /api/subscribe` — used by the quiz, `/free`, and the exit-intent modal. Validates
  email, rate-limits by IP (in-memory, 5/10min — fine at this scale), upserts into
  `leads`, sends an instant welcome email keyed to the source.
- `GET /api/cron/drip` — runs daily via Vercel Cron (`vercel.json`), sends the 5-email
  drip sequence (day 1/2/4/6/8 after signup) to leads who haven't unsubscribed, tracking
  sends in `lead_emails` to avoid duplicates. Protected by `CRON_SECRET`.
- `GET /api/unsubscribe?email=&token=` — one-click unsubscribe link included in every
  email footer, sets `unsubscribed = true` on the lead.
- All templates live in `lib/email.ts` — navy/amber branded, single CTA each, 6th–8th
  grade reading level, no fake urgency.

## SEO content hub (`/guides/*`)

- `content/articles/` — 10 original articles (1,200–1,800 words each) as structured TS
  modules, registered in `content/articles/index.ts`. Each has Article + FAQPage +
  Breadcrumb JSON-LD, a mid-article quiz CTA, an end-of-article bundle CTA, and 2–3
  related-article links.
- `app/guides/page.tsx` — index listing all articles.
- `app/guides/[slug]/page.tsx` — shared article template (statically generated).
- All included in `app/sitemap.ts`; `app/robots.ts` allows crawling everything except
  `/success`, `/trainer`, and `/api/`.

## Analytics

`lib/track.ts` (client) posts lightweight, no-PII events to `POST /api/track`, which
inserts into the Supabase `events` table: `quiz_start`, `quiz_complete` (with score),
`email_submitted` (with source), `checkout_click` (with page location), `article_view`.
No cookies, no PII beyond what's already in `leads`. No-ops silently if Supabase isn't
configured. Swap in Plausible/GA4 later if you want richer analytics — this is the
zero-dependency baseline.

Vercel Analytics (`@vercel/analytics`) is also wired into `app/layout.tsx` for
visitor/pageview data — check the Analytics tab in your Vercel project dashboard.

## Notes

- No fabricated reviews, stats, pass rates, or countdown timers anywhere. Testimonials
  render an honest empty state until real ones exist. Regulation-related claims in the
  SEO articles include "verify current requirements with your state / FMCSA" language
  since rules can change.
- Reading level is 6th–8th grade site-wide, short sentences, friendly/confident tone —
  applies to landing page copy, quiz explanations, emails, and articles alike.
- Sticky mobile CTA bar appears after 50% scroll, hidden on desktop.
- Exit-intent modal is desktop-only, once per session, and never shows on `/quiz` or
  `/success`.
